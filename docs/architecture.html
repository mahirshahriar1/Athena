Athena

Autonomous competitive intelligence platform — multi-agent LangGraph pipeline that researches a company, builds a runtime RAG index, self-critiques, and ships a structured report.

Pipeline

One StateGraph orchestrates the whole run. Parallel fan-out uses the Send API; operator.add and a custom reducer merge results from concurrent branches; a critic loop retries analysts until a quality bar is met; a human-in-the-loop interrupt pauses the graph for plan approval.

Nodes

Planner

planner_node

Asks the LLM for 4–6 concrete research tasks. If a seed URL is provided, prepends a direct-load task and hints the prompt.

Scraper

scraper_worker_node

DuckDuckGo search → load top URLs via WebBaseLoader → LLM extracts structured facts. Seed-URL tasks bypass DDG and load directly.

RAG

rag_ingest_node

RecursiveCharacterTextSplitter (1000/200) chunks each doc, embeds with BGE-small or OpenAI, persists to a per-company Chroma collection.

Analyst

analyst_worker_node

Three roles run in parallel (market / technical / sentiment). Each queries Chroma with a role-specific keyword bag and writes a section.

Critic

critic_node

Scores every section 0–1 on specificity, evidence, and actionability. Routes back to analysts up to 3 times if any score is below 0.75.

Writer

writer_node

Synthesizes all analyst sections plus critique into a Markdown report with executive summary, structured sections, and a sources list.

Comparator

run_comparator

Off-graph: takes two completed reports and writes a side-by-side comparison with a table, per-company wins, and recommendations.

Layers

Frontend

Next.js 14 + Redux Toolkit

App Router, Tailwind dark theme. Redux holds research state; a WebSocket hook streams graph events for the live timeline. React.memo + useCallback throughout.

API

FastAPI

REST for start / approve / status / report / history / compare; WebSocket at /api/ws/research/{job_id} streams astream_events output.

Orchestration

LangGraph StateGraph

Compiled once at startup with MemorySaver and interrupt_before=["scraper_dispatch"]. Threads are keyed by job_id.

LLM

Provider-agnostic via LangChain

get_llm() returns Groq or Anthropic per env var. Both implement BaseChatModel so the rest of the code doesn't care.

RAG

ChromaDB PersistentClient

One collection per company. Same embedding model used for ingest and query (BGE-small default; OpenAI optional).

Persistence

SQLite (stdlib)

data/athena.db stores past runs, comparison metadata, final reports, and token totals. Survives restarts.

Observability

TokenTracker callback

Per-job token accumulator. Surfaces input / output / total / call count in the UI badge and persists with each finished run.

Search

DuckDuckGo (no key)

Free, no API key required. Each scraper worker runs one query and loads the top 3 URLs via WebBaseLoader.

Request lifecycle

  1. POST /api/research/start — graph runs planner, pauses at the interrupt_before on scraper_dispatch. SQLite row inserted.
  2. Frontend shows the plan; user approves.
  3. POST /api/research/{id}/approve — flips plan_approved=True in graph state, returns immediately.
  4. Frontend opens WebSocket. Backend calls astream_events(None, ...) which resumes from the checkpoint and emits node-start / token-stream / node-end events in real time.
  5. Critic may loop back to analysts up to 3×.
  6. Writer emits the final report. WS sends complete with token totals. Report and tokens persist to SQLite.
  7. Frontend fetches GET /api/research/{id}/report and renders the Markdown.
